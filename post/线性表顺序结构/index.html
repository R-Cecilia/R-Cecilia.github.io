线性表（顺序存储结构）

1. 引言

线性表（Linear List）是数据结构中最基础、最常用的逻辑结构。顺序存储结构（Sequential Storage）则是线性表最直观的实现方式，它通过一段连续的内存地址来存储数据。

本文将以一元多项式的项（系数与指数）作为数据元素，展示如何用 C++ 实现一个健壮的顺序表，并重点讨论在开发过程中常见的逻辑陷阱及其修正方案。

2. 核心定义

为了处理多项式，我们将数据元素定义为一个包含系数（coef）和指数（expn）的结构体。

#include <iostream>
using namespace std;

// 最大长度定义
#define MAX_SIZE 100

// 定义多项式的项
struct Term {
    float coef; // 系数
    int expn;   // 指数
};

typedef Term ElemType;

// 定义顺序表结构
struct SqList {
    ElemType *elem; // 存储空间的基地址
    int length;     // 当前长度
};

3. 基本操作实现与逻辑修正

3.1 初始化与销毁（资源管理）

在 C++ 中，必须手动管理堆内存。我们使用 new[] 分配数组，并使用 delete[] 释放，防止内存泄漏。

// 初始化顺序表
bool InitList(SqList &L) {
    L.elem = new ElemType[MAX_SIZE];
    if (!L.elem) return false;
    L.length = 0;
    return true;
}

// 销毁顺序表
void DestroyList(SqList &L) {
    if (L.elem) {
        delete[] L.elem; // 释放数组空间
        L.elem = nullptr;
        L.length = 0;
    }
}

3.2 查找操作（逻辑修复）

修正说明：确保循环遍历完所有已存储元素。只有当整个表遍历结束仍未匹配时，才返回查找失败标识（0）。

// 按指数查找元素位置
int LocateElem(SqList L, int e) {
    for (int i = 0; i < L.length; i++) {
        if (L.elem[i].expn == e) return i + 1; // 返回位序（1-based）
    }
    return 0; // 查找失败
}

3.3 插入操作（边界与溢出保护）

修正说明：

  1. 边界条件：允许在第 1 到 length + 1 个位置插入（length + 1 即在表尾追加）。
  2. 溢出保护：插入前必须判断当前长度是否已达到 MAX_SIZE。
bool ListInsert(SqList &L, int i, ElemType e) {
    // 1. 检查位序是否合法
    if (i < 1 || i > L.length + 1) return false;
    // 2. 检查存储空间是否已满
    if (L.length >= MAX_SIZE) return false;

    // 3. 将第 i 个位置及之后的元素后移
    for (int j = L.length - 1; j >= i - 1; j--) {
        L.elem[j + 1] = L.elem[j];
    }
    L.elem[i - 1] = e; // 插入新元素
    L.length++;
    return true;
}

3.4 删除操作（循环变量修正）

修正说明：修复了原逻辑中可能出现的死循环或越界问题。删除第 i 个元素后，其后的元素需依次前移。

bool ListDelete(SqList &L, int i) {
    // 1. 检查位序合法性
    if (i < 1 || i > L.length) return false;

    // 2. 将第 i 个位置之后的元素前移
    for (int j = i; j < L.length; j++) {
        L.elem[j - 1] = L.elem[j];
    }
    L.length--;
    return true;
}

4. 算法复杂度分析

操作 时间复杂度 说明
GetElem (取值) $O(1)$ 随机存取，通过下标直接定位
LocateElem (查找) $O(n)$ 需要线性扫描全表
ListInsert (插入) $O(n)$ 平均移动 $n/2$ 个元素
ListDelete (删除) $O(n)$ 平均移动 $(n-1)/2$ 个元素

5. 避坑指南：关键逻辑总结

在实现顺序表时，开发者常犯的错误及修正方案如下：

  1. 位序 vs 下标：
    • 位序从 1 开始，数组下标从 0 开始。在代码中操作 L.elem[i-1] 时要保持警惕。
  2. 插入越界：
    • 不要漏掉 i > L.length + 1 的判断，否则会造成数据不连续。
  3. 查找逻辑中断：
    • LocateElem 必须在 for 循环外部返回失败结果，不可在循环内过早返回。
  4. 内存泄露：
    • 明确 new[] 必须对应 delete[]。

6. 后续工作展望

目前的实现是一个基础版本，后续可以在此基础上进行深度优化：

  1. 动态扩容机制：引入类似 std::vector 的策略，当 length == MAX_SIZE 时自动申请 2 倍空间，提升灵活性。
  2. 功能扩展：基于此结构实现两个多项式的相加、相减算法。
  3. 存储对比：实现“链式存储结构”的多项式，对比在处理稀疏多项式时两者的空间利用率。
  4. OOP 封装：将 struct 和全局函数重构成 C++ 类（Class），利用构造与析构函数自动管理资源。

本文旨在通过对顺序表的底层实现，帮助读者理解内存连续分配的重要性及基本算法的边界处理。
