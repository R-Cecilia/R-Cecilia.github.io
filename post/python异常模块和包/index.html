Python—异常,模块和包

1. 异常

异常就是程序运行中出现的错误,即bug.

1.1 异常的捕获

在可能发生异常的地方进行捕获.当异常出现时,提供解决方案,而不是任由其导致程序无法运行.

1.2 捕获异常的语法 

{
    try:
        可能要发生的异常语句
    except[异常 as 别名:]
        出现异常准备的手段
    [else:]
        未出现异常应该做的事情
    [finally:]
        无论是否出现异常都运行的块
    #[]表示内容可以省略,日常应用时,一般try-except就够用了.
}

1.3 异常具有传递性

当所有函数都没有捕获异常时,程序运行就会报错.

2. 模块

2.1 模块的概念

模块就是一个python文件.模块能定义函数,类和变量,模块里也能包含可执行代码.

2.2 模块的导入 

{
    [from 模块名] import [模块|类|变量|函数|*] [as 别名]
}

2.3 自定义模块

2.3.1 导入自定义模块

基本格式同一般模块的导入

2.3.2 导入不同模块的同名功能

以最后一次导入的模块的功能为准

2.3.3 main变量 

{
    if __name__='__main__'
        test()
    '''
    __main__'变量实现了单独运行该程序进行测试，和作为模块调用该程序不受测
    试部分影响的功能
    '''
}

2.3.4 all变量

如果一个文件中有all变量,当使用from … import *导入时,只能导入这个列表中的元素.

3. 包

3.1 包的组成

包是一个文件夹,里面包含了一个init.py文件,该文件夹包含了多个模块文件. 从逻辑上看,包依然是模块

3.2 导入包 

{
    import 包名.模块名[.目标]
    from 包名 import 模块名
}

3.3 通过all控制*导入内容

在init.py文件中写入all语句,控制用*导入包时允许使用的模块.
