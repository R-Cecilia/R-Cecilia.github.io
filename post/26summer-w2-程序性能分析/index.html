程序性能分析

1.基本概念

1.1 程序性能

  • 程序性能是指运行这个程序所需要的内存和时间的多少。
  • 一个程序的空间复杂度是指运行该程序所需要的内存大小。
  • 一个程序的时间复杂度是指运行程序所需要的时间。

1.2 空间复杂度

1.2.1 空间复杂度的组成

  • 指令空间:编译之后的程序指令所需要的存储空间。
  • 数据空间:所有常量和变量值所需要的存储空间。
  • 环境栈空间:环境栈用来保存暂停的函数和方法在恢复运行时所需要的信息

1.3 时间复杂度

1.3.1 时间复杂度的组成

一个程序P所需要的时间是编译时间和运行时间之和。编译时间与实例特征无关。 编译后的程序可以重复运行若干次而无需重新编译。运行时间则常常与实例特征有关。

1.4 多项式计算 

//对于多项式P(x),给定x的值来计算P(x)值
int polyEval(int coeff[],int n,const int &x)
{
    int y=1,value=coeff[0];
    for(int i=1;i<=n;i++)
    {
        y*=x;
        value+=coeff[i]*y;
    }
    return value;
}
//共进行了2n次乘法和n次加法

//利用Horner法则计算多项式P(x)
int polyEval(int coeff[],int n,const int &x)
{
    int value=coeff[n];
    for(int i=1;i<=n;i++)
        value=value*x+coeff[n-i];
    return value;
}
//共进行n次加法和n次乘法

1.5 渐近记法

渐近记法描述的大实例特征的时间或者空间复杂度。

  • 表示法f(n)=O(g(n))代表f(n)渐近小于或等于g(n)。
  • 在记法f(n)=O(g(n))中，除f(n)=0以外，函数g(n)习惯上是单位项，即系数为1的单项，而且通常是令f(n)=O(g(n))为真的最小单位项。
